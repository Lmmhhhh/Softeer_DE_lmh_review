
# W3M4 팀 활동: Predefined Label을 사용하지 않는 감성 분류 방법
1팀 팀원 - 이승연, 이민하, 박민서, 홍준성

## 팀 활동 주제

> `'predefined keywords'가 아닌 다른 방법으로 나누고자 한다면 어떤 방법이 있을까요? 팀과 함께 논의해서 다른 방법을 시도해 보세요.`

이번 과제에서는 [Sentiment140 데이터셋](https://www.kaggle.com/datasets/kazanova/sentiment140)에 미리 부여된 감성 라벨을 이용해 트윗을 분류하였다.

데이터셋에 포함된 기존 라벨을 그대로 사용하는 방법 외에, 트윗 원문을 직접 분석하여 감성을 새롭게 분류할 수 있는 방법을 논의하였다.

---

## 1. 감성 사전 기반 분석

감성 사전에 단어별 긍정 또는 부정 점수가 저장되어 있음. 

대표적인 감성 사전

* VADER
* AFINN
* SentiWordNet

문장에 포함된 단어의 감성 점수를 합산하여 최종 감정 결정 가능 

### 장점

* 기존 데이터셋의 라벨을 사용하지 않아도 된다.
* 별도의 모델 학습 없이 사용할 수 있다.
* 구현이 비교적 간단하다.
* 짧은 SNS 문장 분석에 적합하다.

### 단점

* 사전에 없는 신조어나 은어를 처리하기 어렵다.
* 문장 전체의 복잡한 문맥을 완전히 이해하기 어렵다.

---

## 2. 머신러닝 기반 감성 분류

감성이 라벨링된 별도의 학습 데이터를 이용하여 분류 모델을 직접 학습하는 방법

트윗을 `TF-IDF` 또는 `CountVectorizer`로 벡터화한 후 머신러닝 기반 모델 학습

대표적인 머신러닝 모델
* Naive Bayes
* Logistic Regression
* Support Vector Machine
* Random Forest

### 장점

* Sentiment140의 기존 라벨을 사용하지 않고 감성을 새로 추론할 수 있다.
* 학습 데이터로부터 다양한 감성 표현을 학습할 수 있다.
* 비교적 가벼운 모델로 대량 데이터를 처리할 수 있다.

### 단점

* 별도의 라벨링된 학습 데이터가 필요하다.
* 학습 데이터의 품질과 양에 따라 성능이 달라진다.
* 복잡한 문맥이나 새로운 표현에는 한계가 있을 수 있다.

---

## 3. 문장 임베딩 기반 분석

트윗 문장을 의미를 가진 벡터로 변환한 후 감성을 분류하는 방법

대표적인 임베딩 모델

* Word2Vec
* GloVe
* FastText
* Sentence-BERT

### 장점

* 단순한 단어 일치보다 문장의 의미를 더 잘 반영한다.
* 서로 다른 단어를 사용한 유사 표현을 함께 처리할 수 있다.
* 기존 라벨이 없는 문장에도 적용할 수 있다.

### 단점

* TF-IDF보다 계산량이 많다.
* 임베딩 모델 외에 별도의 분류 모델이 필요하다.
* 임베딩만으로 감성이 자동으로 결정되는 것은 아니다.

---

## 4 Transformer 기반 감성 분석

Transformer 기반의 사전 학습 감성 분석 모델을 사용

대표적인 감성 분석 모델
* BERT
* RoBERTa
* DistilBERT
* Twitter-RoBERTa

Transformer 모델은 특정 단어의 포함 여부만 확인하지 않고, 문장 전체의 문맥을 함께 고려

### 장점

* 문장 전체의 문맥을 반영할 수 있다.
* 부정 표현과 복합적인 문장을 처리할 수 있다.
* 감성 분석 모델을 처음부터 학습하지 않아도 된다.
* 기존 데이터셋의 라벨 없이 텍스트만으로 감성을 추론할 수 있다.

### 단점

* 모델 크기가 크고 실행 속도가 느릴 수 있다.
* 대량 데이터 처리 시 CPU와 메모리 사용량이 증가한다.

---

## 5. LLM 기반 감성 분류

GPT와 같은 대규모 언어 모델에 트윗을 입력하고 감성을 판단하도록 하는 방법

LLM은 감성 라벨뿐만 아니라 판단 이유도 함께 제시해줌.

### 장점

* 복합 감정과 문맥을 비교적 잘 이해할 수 있다.
* 감성 판단의 이유를 확인할 수 있다.
* 별도의 모델 학습이 필요하지 않다.

### 단점

* API 비용이 발생할 수 있다.
* 대량 데이터 처리 시 시간과 비용이 증가한다.
* MapReduce 환경에 직접 적용하기에는 외부 API 호출 부담이 크다.

---

## 6. Clustering 활용한 방법

감성 라벨을 사용하지 않고 비슷한 트윗끼리 자동으로 묶는 비지도 학습 방법

트윗을 TF-IDF 또는 문장 임베딩으로 변환한 뒤, K-Means 등의 군집화 알고리즘을 적용

```text
트윗 원문
    ↓
TF-IDF 또는 문장 임베딩
    ↓
K-Means 군집화
    ↓
비슷한 문장끼리 그룹 생성
```

### 장점

* 미리 정의된 라벨이나 학습 데이터가 필요하지 않다.
* 데이터 내부의 새로운 패턴을 발견할 수 있다.

### 단점

* 감성이 아니라 주제에 따라 군집이 형성될 수 있다.
* 각 클러스터의 의미를 사람이 직접 해석해야 한다.
* 긍정, 중립, 부정으로 명확히 분리된다는 보장이 없다.

---

## 7. 이모지를 활용한 감성 분류

트위터에서는 사용자가 텍스트 대신 이모지로 감정을 표현하거나, 이모지로 감정의 강도를 강조하는 경우가 많음
긍정적 이모지 / 부정적 이모지를 분류하고, 이를 기반으로 문장 분류

### 장점

* 짧은 트윗에서 부족한 감성 정보를 보완할 수 있다.
* 텍스트에 직접 표현되지 않은 감정을 확인할 수 있다.
* 감정의 강도를 추가로 나타낼 수 있다.
* SNS 데이터의 특성을 반영할 수 있다.

### 단점

* 이모지의 의미는 문맥에 따라 달라질 수 있다.

---

## 최종 선정 방법: Twitter-RoBERTa

팀에서는 트위터 데이터에 특화된 Transformer 모델인 `Twitter-RoBERTa`를 사용해 감성을 새롭게 분류하기로 하였다.

* 모델: [cardiffnlp/twitter-roberta-base-sentiment](https://huggingface.co/cardiffnlp/twitter-roberta-base-sentiment)
* 기반 모델: RoBERTa-base
* 사전 학습 데이터: 약 5,800만 개의 트윗
* 분류 결과:

  * Negative
  * Neutral
  * Positive

Twitter-RoBERTa 역시 감성이 라벨링된 데이터를 이용해 학습된 지도학습 모델이다.

하지만 이번 과제에서 사용한 Sentiment140의 `target` 값을 직접 참조하지 않고, 트윗 원문을 모델에 입력해 새로운 감성 결과를 추론한다는 점에서 기존 방식과 차이가 있다.

### Twitter-RoBERTa를 선정한 이유

1. 분석 대상과 같은 트위터 데이터를 기반으로 학습되었다.
2. 트윗의 문맥을 고려하여 감성을 판단할 수 있다.
3. 별도의 모델 학습 없이 사용할 수 있다.
4. Sentiment140의 기존 라벨 없이 트윗 원문만으로 감성을 추론할 수 있다.

---

## 실험 방법

Sentiment140 데이터셋 160만 건 데이터 중 1,000건을 샘플링했다.

Sentiment140의 기존 `target` 값은 사용하지 않고, 트윗 원문만 Twitter-RoBERTa 모델에 입력하였다.


---

## 실험 결과

| 카테고리 | 키워드 방식 | BERT 방식 |
|---|---|---|
| negative | 122 (12.2%) | 332 (33.2%) |
| neutral | 630 (63.0%) | 302 (30.2%) |
| positive | 248 (24.8%) | 366 (36.6%) |

기존 키워드 기반 방식에서는 전체 데이터의 63.0%가 중립으로 분류되어 결과가 한 범주에 치우치는 경향이 나타났다. 
반면 Twitter-RoBERTa 방식에서는 비교적 고르게 분류되었다.

---

## 추가 토의: 신조어와 비꼬는 표현

Twitter-RoBERTa와 같은 감성 분석 모델도 모든 트윗을 완벽하게 분류하지는 못한다.

팀에서는 특히 다음과 같은 표현을 어떻게 처리할지 추가로 논의하였다.

* 신조어
* 인터넷 은어
* 축약어
* 반어법
* 비꼬는 표현
* 긍정과 부정이 함께 포함된 문장

---
### 신조어를 중심으로 주변 문장 문맥 파악 
신조어 주변에 함께 등장하는 단어와 문장 전체의 흐름을 분석하여 추론하는 방법 고려

### Sarcasm Detection 모델 활용

비꼬는 표현을 처리하기 위한 모델을 함께 사용하는 아이디어 제시

참고 자료:

* [Sarcasm Detection Using Deep Convolutional Neural Networks](https://www.themoonlight.io/ko/review/sarcasm-detection-using-deep-convolutional-neural-networks-a-modular-deep-learning-framework)

Sarcasm Detection 모델은 입력 문장이 일반적인 표현인지, 비꼬는 표현인지를 별도로 판단한다.

### 장점

* 표면적인 긍정 표현과 실제 부정 의도를 구분하는 데 도움이 된다.
* 반어법이 자주 사용되는 SNS 분석에 적합하다.
* 일반 감성 분석 모델의 오분류를 보완할 수 있다.

### 단점

* 두 개의 모델을 실행하므로 처리 시간이 증가한다.
* Sarcasm Detection 모델도 항상 정확한 것은 아니다.
* 두 모델의 결과가 충돌할 때 적용할 기준이 필요하다.
* 감성 결과를 조정하기 위한 추가 규칙이나 가중치가 필요하다.

---

## 팀 토의 결론

팀에서는 기존 라벨을 직접 사용하지 않는 방법으로 다음 방식을 논의하였다.

* 감성 사전
* 머신러닝
* 문장 임베딩
* Transformer
* LLM
* 군집화
* 이모지 분류

그중 과제 데이터가 트위터 댓글이라는 점을 고려하여, 약 5,800만 개의 트윗으로 사전 학습된 Twitter-RoBERTa 모델을 사용해 트윗의 감성을 새롭게 분류하였다.

Twitter-RoBERTa는 Sentiment140의 기존 `target` 값을 사용하지 않고 트윗 원문을 바탕으로 Negative, Neutral, Positive를 추론할 수 있다.

1,000개의 트윗을 분석한 결과, 기존 정의된 키워드 기반 방식은 중립으로 분류된 비율이 가장 높았지만, 
Twitter-RoBERTa 방식은 비교적 고르게 분류되었다. 이를 통해 사전 학습 모델이 키워드 방식보다 다양한 감성 표현과 문맥을 반영할 가능성이 있음을 확인하였다.

다만 사전 학습 모델을 사용했다고 해서 결과가 반드시 정확한 것은 아니다. 신조어, 반어법, 비꼬는 표현 처럼 문맥에 따라 의미가 달라지는 표현은 여전히 잘못 분류될 수 있다.

이를 보완하기 위해 팀에서는 다음과 같은 확장 방법을 논의하였다.

1. 기본 감성 분류에는 Twitter-RoBERTa를 사용한다.
2. 비꼬는 표현은 Sarcasm Detection 모델로 별도 분석한다.
5. 일부 데이터를 사람이 직접 라벨링하여 모델 결과의 정확도를 검증한다.

최종적으로 팀에서는 Sentiment140의 기존 라벨을 그대로 사용하는 방식보다 트윗 원문을 직접 분석하고, 비꼼과 이모지 정보를 함께 활용하는 방식이 새로운 데이터에 적용하기 쉽고 문맥을 더 잘 반영할 수 있다고 판단하였다.
