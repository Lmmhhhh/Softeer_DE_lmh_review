# 데이터 기반 의사결정과 데이터 엔지니어링

## 1. 데이터 기반 의사결정

데이터를 활용할 때는 **데이터 만능주의에 빠지지 않도록 주의**

모든 의사결정을 데이터에만 의존하기보다, 해결하려는 문제가 무엇인지 먼저 정의하는 것이 중요

의사결정 방식

> 
> 
> 
> 
> | 구분 | 설명 |
> | --- | --- |
> | Heuristic | 경험과 직관에 기반한 판단 |
> | Evidence-based | 근거와 데이터에 기반한 판단 |

데이터 기반 의사결정에서 중요한 것은 단순히 데이터를 많이 모으는 것이 아니라,

⭐️⭐️⭐️⭐️⭐️ **“무슨 문제를 해결해야 하는가?”** 를 명확히 ⭐️⭐️⭐️⭐️⭐️

---

## 2. Actionable Data

좋은 데이터는 단순히 저장되어 있는 데이터가 아니라, 실제 행동과 의사결정으로 이어질 수 있는 **Actionable Data**

ex) 광고 성과 지표

| 지표 | 의미 |
| --- | --- |
| CPM | Cost Per Mille, 1,000회 노출당 비용 |
| CPC | Cost Per Click, 클릭당 비용 |
| CTR | Click Through Rate, 노출 대비 클릭률 |
| STR | 구매자 발생 시에만 광고료를 지불하는 방식 |

같은 영화 광고라도 **장르, 인지도, 타깃 고객에 따라** 효율적인 마케팅 방식은 달라질 수 있음

그러므로 데이터를 해석할 때는 단순 수치뿐 아니라 **비즈니스 도메인에 대한 이해**가 함께 필요

---

## 3. 비즈니스 도메인 파악의 중요성

데이터를 잘 활용하기 위해서는 비즈니스 도메인을 이해하는 것이 중요

> 지금 이 상황에서 무엇이 중요한가?
> 

데이터 자체보다 중요한 것은

**데이터를 통해 어떤 문제를 해결하고, 어떤 가치를 만들 수 있는지 판단하는 능력**

---

## 4. 데이터 관련 역할

### 데이터 아키텍트

> 데이터의 전체 구조와 흐름을 설계
> 
> 
> 즉, 데이터를 어떻게 저장하고 연결하고 활용할지에 대한 **청사진** 그리는 역할
> 

### 데이터 엔지니어

> 데이터 아키텍처를 바탕으로 raw data를 분석과 의사결정에 사용할 수 있는 형태로 가공
> 
> 
> 원천 데이터를 수집하고 정제하여 실제로 활용 가능한 데이터 파이프라인을 구축하는 역할
> 

---

## 5. 문제 해결 방식

문제를 해결할 때는 처음부터 완벽한 답을 찾으려 하기보다, 가장 적은 비용으로 빠르게 시도하고 피드백을 받는 방식이 효과적

- 가장 적은 비용으로 빠르게 해결할 방법 찾기
- 자주, 여러 번 피드백 받기
- Step by Step으로 테스트하기
- 한 단계가 통과되면 다음 단계로 넘어가기
- **Trial and Error**를 통해 개선하기

> 프롬프트를 작성할 때도 마찬가지
> 
> 
> 한 번에 모든 것을 요청하기보다, 여러 단계로 나누어 요청하고 각 단계마다 결과를 검토한 뒤 다음 단계로 넘어가는 좋음
> 
> 또한 AI에게 답변을 요청할 때는 항상 **근거를 함께 제시하도록 요청하는 습관**이 필요
> 

---

## 6. 데이터 엔지니어에게 필요한 역량

- 데이터 엔지니어는 단순히 데이터베이스나 SQL만 아는 것이 아니라, 데이터가 흘러가는 전체 시스템을 이해해야 함
    
    [OS, 인프라, 가상머신, 네트워크, 응용, 데이터베이스, 데이터 웨어하우스, 데이터 파이프라인, etc.]
    
- 비즈니스 요구사항을 기술적 명세로 바꾸는 능력도 중요

---

## 7. Product Lifecycle

데이터 엔지니어도 제품의 전체 생애주기를 이해하고 경험해보는 것이 좋음

```
아이디에이션 → 개발 → 출시 → 운영 → 분석 → 최적화
```
